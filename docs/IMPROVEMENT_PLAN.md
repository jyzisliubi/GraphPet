# GraphPet 项目锐评与改进计划

> 锐评日期：2026-07-02
> 当前版本：v0.1.0 (开发中)

---

## 一、项目概览

GraphPet 是一个基于 Electron + React + Python 的桌面AI宠物应用，核心特色是：
- 🐱 Live2D 桌宠 Nito（可换肤，支持点击互动）
- 🧠 知识图谱喂食系统（喂文件→三元组提取→长期记忆）
- 💬 AI 多轮对话（支持 Ollama/DeepSeek/智谱/硅基流动/OpenAI 等）
- 📊 知识图谱可视化
- 🎯 成长系统（智力等级、XP）

---

## 二、诚实锐评

### ✅ 做得好的地方

1. **概念有差异化**："喂文件给桌宠学习"+"知识图谱记忆"在桌宠赛道有独特性，不是纯聊天套壳
2. **技术栈选择合理**：Electron+React+TypeScript+Python FastAPI 分层清晰
3. **功能覆盖广**：拖放喂食、URL喂食、批量喂食、换肤、右键菜单、主动对话、图谱可视化、吐掉记忆、三元组预览
4. **多LLM支持**：覆盖本地(Ollama)和云端(DeepSeek/智谱/月之暗面/硅基流动/OpenAI)
5. **RAG三元组质量现在达标了**：中文prompt+JSON模式+关键词优先策略，关系从冗长英文句变为2-6字中文谓词

## 三、GitHub热门项目对比分析

| 项目 | Stars | 核心优势 | GraphPet差距 |
|------|-------|---------|-------------|
| **Open-LLM-VTuber** | 9.5k | Vtuber形态+TTS/ASR/语音打断+多模型支持 | 缺语音、形态不贴近桌面 |
| **Shimeji-ee** | 5k+ | 自由走动/爬边/掉落/拖拽/多宠物互动 | 宠物静止不动 |
| **Live2DViewerEX** | 很多用户 | 模型管理/表情/动作丰富/壁纸模式 | 模型交互单一 |
| **Character.AI** | - | 沉浸式对话/角色记忆/长期关系 | 对话体验差 |
| **obsidian-releases** | - | 图谱可视化标杆 | 图谱太粗糙 |

**核心差距总结**：
1. **交互不够"活"**：热门桌宠宠物会动、会走、会做各种动作，你的Nito大部分时间是静止idle
2. **对话体验不够"沉浸"**：热门AI陪伴产品对话是核心体验，你的对话像附加功能
3. **门槛太高**：用户打开就能玩，不需要配置API Key
4. **完成度不够**：UI打磨、错误处理、边界情况都差一口气

## 六、技术债务与注意事项

1. **Python后端进程管理**：Electron退出时需要确保Python子进程被正确kill（当前已有处理但需要验证Windows下是否完全干净）
2. **pixi-live2d-display**：这个库维护不太活跃，注意版本锁定
3. **LightRAG**：自定义了prompt，升级LightRAG版本时需要验证prompt兼容性
4. **无git提交**：项目还没有初始commit，建议尽快git init并做好版本管理
5. **Python环境**：当前依赖系统Python（miniconda），打包时需要内嵌Python
